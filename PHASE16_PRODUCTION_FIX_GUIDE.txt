PHASE 16 — PRODUCTION AUTH + STRIPE FIX PACK

This pack addresses 3 practical issues:

1. Header branding
- The site header now reads:
  NEXT_PUBLIC_APP_NAME
- Optional logo path:
  NEXT_PUBLIC_LOGO_PATH=/logo-mark.svg
- You can change the text and logo without editing code again.

2. Supabase email branding
- You can change the email sender flow away from the default Supabase look.
- In Supabase:
  Authentication -> Email
  - Edit Email Templates
  - Configure Custom SMTP when ready
- Recommended production route:
  a) replace template branding first
  b) later connect custom SMTP (Resend / SendGrid / Mailgun)

3. Stripe upgrade / auto logout / not turning Pro
This is usually production configuration, not front-end code.
Check these in order:

A. Vercel Environment Variables
- NEXT_PUBLIC_SITE_URL=https://YOUR-REAL-VERCEL-DOMAIN
- STRIPE_SECRET_KEY=correct test/live key
- STRIPE_PRICE_ID_PRO_MONTHLY=matching key mode
- STRIPE_WEBHOOK_SECRET=the webhook signing secret for THIS environment

B. Supabase Authentication -> URL Configuration
- Site URL = https://YOUR-REAL-VERCEL-DOMAIN
- Add Redirect URLs:
  https://YOUR-REAL-VERCEL-DOMAIN/auth/reset-password
  https://YOUR-REAL-VERCEL-DOMAIN/auth/login
  https://YOUR-REAL-VERCEL-DOMAIN/dashboard
  https://YOUR-REAL-VERCEL-DOMAIN/pricing

C. Stripe Webhook Endpoint
- Must point to:
  https://YOUR-REAL-VERCEL-DOMAIN/api/stripe/webhook
- Recommended events:
  checkout.session.completed
  customer.subscription.updated
  customer.subscription.deleted

D. Stripe Test Card
- 4242 4242 4242 4242 is SUPPOSED to succeed in test mode.
- That is normal.
- If payment succeeds but plan stays Free, the problem is webhook/config sync.

E. If returning from Stripe logs you out
- This almost always means Supabase Site URL / redirect URLs are not fully set for production.

Recommended order:
1. Replace header branding
2. Update Vercel env
3. Update Supabase auth URLs
4. Update Stripe webhook endpoint
5. Redeploy
6. Test signup -> login -> upgrade again
